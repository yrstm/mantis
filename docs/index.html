Rendered pages to agent Markdown

Mantis is a browser extension that captures the page you can see, filters page chrome, and copies clean Markdown for agents.

It runs over the live DOM, so it works after client-side rendering and logged-in state. Nothing is uploaded by default.

# Pricing API Guide

The Pricing API returns the current plan, metered usage, and renewal date for an account.

Requests must include a bearer token with the `billing:read` scope.

Install

1. Get the repo 
git clone https://github.com/yrstm/mantis.git
cd mantis
2. Load in Brave

Open brave://extensions, enable Developer mode, choose Load unpacked, and select the repo root.

3. Capture a page

Pin Mantis, open a web page, and click the toolbar button. Select text first to capture only the selected DOM range.

No local server is needed for the extension. The local demo server is only for development.

Why not normal copy?

Browser copy brings navigation, sidebars, footer text, and no source metadata. Mantis keeps the article-shaped content and emits metadata an agent can use.

Screenshot API

The browser extension captures DOM. Screenshot capture is a separate local-helper workflow. The library API, Mantis.fromImage(), lets a tool pass OCR or vision output into the same Markdown pipeline without adding OCR dependencies to the extension.
